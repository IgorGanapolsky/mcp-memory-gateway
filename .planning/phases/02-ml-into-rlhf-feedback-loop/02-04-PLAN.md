  <behavior>
SEQUENCE TRACKING TEST CASES (tests/feedback-sequences.test.js):

1. accepted captureFeedback() creates feedback-sequences.jsonl:
   input: captureFeedback({ signal:'positive', context:'test', whatWorked:'worked', tags:['testing'] })
   expected: feedback-sequences.jsonl file exists in RLHF_FEEDBACK_DIR

2. sequence entry has correct schema:
   expected entry fields: id (string starting with 'seq_'), timestamp (ISO string), targetReward (1 for positive), targetTags (array), features (object), label ('positive')

3. targetReward is 1 for positive signal:
   expected: entry.targetReward === 1

4. targetReward is -1 for negative signal:
   input: captureFeedback({ signal:'negative', context:'ctx', whatWentWrong:'bad', whatToChange:'fix it', tags:['debugging'] })
   expected: entry.targetReward === -1

5. features.rewardSequence is an array:
   expected: Array.isArray(entry.features.rewardSequence)

6. features.tagFrequency is an object:
   expected: typeof entry.features.tagFrequency === 'object'

7. rejected captureFeedback() does NOT create sequence entry:
   input: captureFeedback({ signal:'up' })  // invalid signal → rejected
   expected: feedback-sequences.jsonl either absent or has 0 lines

8. Multiple accepted calls append multiple sequence entries:
   input: 3 accepted captureFeedback() calls
   expected: feedback-sequences.jsonl has 3 lines

DIVERSITY TRACKING TEST CASES (tests/diversity-tracking.test.js):

1. accepted captureFeedback() creates diversity-tracking.json:
   expected: diversity-tracking.json exists in RLHF_FEEDBACK_DIR

2. diversity-tracking.json has diversityScore field:
   expected: typeof diversity.diversityScore !== 'undefined'

3. diversityScore is a numeric value:
   expected: Number(diversity.diversityScore) >= 0 && Number(diversity.diversityScore) <= 100

4. domains object is populated:
   expected: typeof diversity.domains === 'object' && Object.keys(diversity.domains).length > 0

5. domain count matches feedback domain:
   input: captureFeedback() with tags=['testing'] → domain inferred as 'testing'
   expected: diversity.domains.testing.count >= 1

6. lastUpdated is set:
   expected: typeof diversity.lastUpdated === 'string'

7. diversityScore updates after second feedback:
   input: 2 accepted captureFeedback() calls with different domains
   expected: diversity.lastUpdated changes between the two reads

8. diversityScore is not NaN or Infinity on first entry (edge case from Research pitfall #4):
   input: fresh tmpdir, single captureFeedback() call
   expected: isFinite(Number(diversity.diversityScore))

BOTH TEST FILES use:
- process.env.RLHF_FEEDBACK_DIR = tmpDir isolation
- os.tmpdir() + fs.mkdtempSync for temp dir
- fs.rmSync(tmpDir, { recursive: true }) cleanup in finally
- delete process.env.RLHF_FEEDBACK_DIR in finally
