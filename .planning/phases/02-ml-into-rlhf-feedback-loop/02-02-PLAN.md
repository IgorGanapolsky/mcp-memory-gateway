STEP 2: Add helper functions (insert before captureFeedback function definition):

a) inferDomain(tags, context) — keyword-based domain inference:
   - tagSet = new Set((tags || []).map(t => t.toLowerCase()))
   - ctx = (context || '').toLowerCase()
   - Check in order: test→'testing', security/secret→'security', perf/performance→'performance',
     ui/component→'ui-components', api/endpoint→'api-integration', git/commit→'git-workflow',
     doc/readme→'documentation', debug/error→'debugging', arch/design→'architecture',
     data/schema→'data-modeling'
   - Return 'general' if no match

b) buildSequenceFeatures(recentEntries, currentEntry):
   - sequence = [...recentEntries, currentEntry]
   - rewardSequence: sequence.map(f => f.signal === 'positive' ? 1 : -1)
     NOTE: rlhf uses f.signal ('positive'/'negative') NOT f.reward (1/-1) — Subway schema differs
   - tagFrequency: reduce over sequence, counting (f.tags || []) occurrences
   - recentTrend: calculateTrend(sequence.slice(-5).map(f => f.signal === 'positive' ? 1 : -1))
   - timeGaps: calculateTimeGaps(sequence) using parseTimestamp (require from ./feedback-schema)
   - actionPatterns: extractActionPatterns(sequence) counting tag positive/negative occurrences
   - Return { rewardSequence, tagFrequency, recentTrend, timeGaps, actionPatterns }

c) calculateTrend(rewards):
   - if length < 2: return 0
   - recent = rewards.slice(-3)
   - return recent.reduce((a,b) => a+b, 0) / recent.length

d) calculateTimeGaps(sequence):
   - Import parseTimestamp from './feedback-schema' (already required in file? If not, add require)
   - For i=1..length-1: compute (curr - prev) / 1000 / 60 (minutes)
   - Skip pairs where either timestamp is null
   - Return array of gap values

e) extractActionPatterns(sequence):
   - For each entry, for each tag: increment patterns[tag].positive or patterns[tag].negative
   - Return patterns object

f) appendSequence(feedbackEvent, paths):
   - sequencePath = path.join(paths.FEEDBACK_DIR, 'feedback-sequences.jsonl')
   - recent = readJSONL(paths.FEEDBACK_LOG_PATH).slice(-SEQUENCE_WINDOW)
     (readJSONL is already defined in this file — call it directly)
   - features = buildSequenceFeatures(recent, feedbackEvent)
   - entry = {
       id: 'seq_' + Date.now(),
       timestamp: new Date().toISOString(),
       targetReward: feedbackEvent.signal === 'positive' ? 1 : -1,
       targetTags: feedbackEvent.tags,
       features,
       label: feedbackEvent.signal === 'positive' ? 'positive' : 'negative',
     }
   - appendJSONL(sequencePath, entry)  (appendJSONL already defined in this file)

g) updateDiversityTracking(feedbackEvent, paths):
   - diversityPath = path.join(paths.FEEDBACK_DIR, 'diversity-tracking.json')
   - Load existing or use default: { domains: {}, lastUpdated: null, diversityScore: 0 }
   - domain = inferDomain(feedbackEvent.tags, feedbackEvent.context)
   - Init domain entry if absent: { count:0, positive:0, negative:0, lastSeen:null }
   - Increment count, lastSeen, positive or negative
   - Compute variance: totalFeedback = sum of all domain counts; idealPerDomain = totalFeedback / DOMAIN_CATEGORIES.length
   - variance = sum of (d.count - idealPerDomain)^2 / Math.max(domainCount, 1)
     NOTE: use Math.max(domainCount, 1) to avoid div-by-zero on first entry
   - diversityScore = Math.max(0, 100 - Math.sqrt(variance) * 10).toFixed(1)
   - recommendation: score < 50 → suggest uncovered domains; else 'Good diversity'
   - diversity.lastUpdated = new Date().toISOString()
   - fs.writeFileSync(diversityPath, JSON.stringify(diversity, null, 2) + '\n')
     (fs is already required in this file)
